Let's build

I'm a physician who codes. I build clinical tools because I needed them and the ones that existed were slow, bloated, paywalled, or tracking users.

If you have a problem in medicine, education, or public health that software could solve — I want to hear about it. Especially interested in projects that respect patient privacy, reduce cognitive load, or improve access to accurate information.

get in touch →
> working principles
✓
Open source by default. Code should be auditable.
✓
No trackers, no ads. Non-negotiable.
✓
Fast and small. If it needs a 2MB bundle it's probably wrong.
✓
Nothing here is medical advice. All tools are educational.
✓
S.D.G. — work done for a reason beyond the resume.
> what I'm looking for
01/
Clinical Tool Development
Calculator, simulator, decision-support, or patient education resource. I build fast, deploy to GitHub Pages or Vercel. Zero infrastructure cost.
02/
Medical Education Projects
Curriculum supplementation, interactive cases, classroom simulation. Rhesus and CaseRunner are proof of concept — they can be extended.
03/
Privacy-First Health Tech
If your project avoids the surveillance economy, we're already aligned. HIPAA-compliant client-side tools, local-first architecture, no tracking.
04/
Clinical Accuracy Review
Code review for health software. I can assess whether what you've built makes medical sense before it touches patients.
05/
Collaborative Research Tools
Data visualization, clinical informatics, open-access research infrastructure. If it serves patients or clinicians, I'm interested.
